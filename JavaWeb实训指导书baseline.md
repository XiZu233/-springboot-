# JavaWeb 实训指导书（零基础版）

## 一、实训目标

通过完成一个完整的“图书管理系统”，零基础学习者能够：

1. 理解 Web 应用的基本工作方式：浏览器 → 服务器 → 数据库 → 返回页面。
2. 掌握 Spring Boot 快速搭建 Web 项目的流程。
3. 理解 Spring Security 登录认证、角色授权的基本概念。
4. 理解 Thymeleaf 模板引擎如何渲染 HTML 页面。
5. 理解 MyBatis 如何实现 Java 代码与 SQL 的映射。
6. 学会使用 AI Agent（如 Claude Code）辅助阅读、修改和验证代码。

## 二、前置知识

本次实训假设你对以下概念有最基础的了解：

- 会安装软件（JDK、MySQL、IDEA）。
- 知道什么是 HTML 表单、超链接、表格。
- 知道 Java 是一种编程语言，MySQL 是一种数据库。
- 不强制要求会写 SQL 或 Java，但需要有耐心跟着步骤操作。

## 三、技术栈说明

| 层级 | 技术 | 作用 |
|------|------|------|
| 后端框架 | Spring Boot 3.2 | 项目骨架，负责接收请求、处理业务 |
| 安全认证 | Spring Security 6 | 登录、注册、权限判断 |
| 页面模板 | Thymeleaf | 把后端数据渲染成 HTML 页面 |
| 数据持久化 | MyBatis | 让 Java 方法直接对应 SQL 语句 |
| 数据库 | MySQL 8.0 | 存储用户和图书数据 |
| 前端样式 | Bootstrap 5 | 让页面美观、响应式 |
| 构建工具 | Maven | 管理依赖、编译打包 |
| JDK | 17+ | Java 运行环境 |

## 五、原始项目功能

在改造之前，系统已实现：

1. **用户注册与登录**：使用 Spring Security 验证账号密码。
2. **图书新增**：填写表单后保存到数据库。
3. **图书列表**：展示所有图书。
4. **图书删除**：删除指定图书。
5. **默认管理员**：启动时自动创建 `admin/123456`。

可以看出，原始项目是一个“能跑起来”的最小系统，但缺少很多真实系统常见的能力：

- 没有区分管理员和普通用户。
- 不能修改图书。
- 不能查看图书详情。
- 没有搜索和分页。
- 没有服务端校验。
- 出错时直接暴露异常信息。

本次实训就是要在原始项目基础上，逐步补齐这些能力。

## 六、如何用 AI Agent 理解代码仓库

对于零基础学习者，直接阅读一个完整项目可能会感到困难。可以借助 AI Agent 按照以下方式提问：

### 6.1 先让 Agent 介绍整体结构

```text
请帮我介绍这个 JavaWeb 项目的目录结构和每一层的作用。
```

### 6.2 逐层深入

```text
请解释 controller、service、mapper、entity 各自是做什么的，并以 BookController 为例说明一个请求的处理流程。
```

### 6.3 追踪一个具体功能

```text
请帮我追踪“用户登录”功能的完整调用链：从浏览器输入用户名密码开始，到数据库验证，再到登录成功跳转。
```

### 6.4 让 Agent 解释你不理解的代码

```text
请解释 SecurityConfig.java 中的 SecurityFilterChain 是如何工作的。
```

### 6.5 让 Agent 帮你做修改并解释

```text
我想给图书列表增加一个搜索框，请告诉我需要改哪些文件，并逐步帮我实现。
```

### 使用 Agent 的原则

1. **先问整体，再问局部**：不要一上来就问某一行代码是什么意思。
2. **要求给出文件路径和行号**：方便你自己打开对照。
3. **要求解释“为什么这样改”**：不要只接受代码，要理解设计原因。
4. **自己验证**：Agent 给出的结果要通过运行项目、查看页面来确认。

## 八、常见问题

### Q1：运行项目时报数据库连接失败？

检查 [application.yml](src/main/resources/application.yml) 中的数据库地址、用户名、密码是否与本地 MySQL 一致。

### Q2：为什么页面显示乱码？

确保 MySQL 数据库和表的字符集为 `utf8mb4`，且 `application.yml` 连接串中包含 `useUnicode=true&characterEncoding=utf-8`。

### Q3：Agent 修改后代码不能运行？

AI 生成的代码可能有误。每次修改后都要：

1. 运行 `mvn clean test` 或 `mvn clean compile`。
2. 启动项目并手动验证对应功能。
3. 把错误信息完整复制给 Agent，要求它排查。
